# Claude Code Prompt: Vibe Coding Web 框架选型 Slides

## 项目背景

我需要制作一个关于 **"Vibe Coding 下 Web 前端框架选型指南"** 的演示文稿（Slides），用于向开发者介绍在 AI 驱动开发时代如何选择合适的前端框架。

**核心观点**：2026 年，Web 开发范式从"手写代码"转向"意图驱动开发"（Vibe Coding），框架选型的标准也随之改变。

---

## 技术要求

### 基础架构
- 使用纯 HTML + CSS + JavaScript 实现
- 单个 `index.html` 文件，可以拆分 CSS 和 JS 文件
- 支持键盘左右箭头切换幻灯片
- 支持底部进度条或页码指示器
- 响应式设计，适配 16:9 屏幕比例

### 视觉风格（Apple Keynote 极简风格，必须严格遵循）
## 🎨 整体视觉风格建议（Apple Keynote 极简风格）

### 背景
- **首选**：纯白色 `#FFFFFF` 或极浅灰 `#F5F5F7`
- **分隔页**：少量深色幻灯片可用于章节分隔（黑色或深灰）
- **禁止**：渐变背景、复杂纹理

### 字体
- **字体家族**：仅使用无衬线字体
  - 推荐：SF Pro / Inter / Helvetica Neue / PingFang SC（中文）
- **标题**：超大字号，加粗，简短有力（建议 48-72pt）
- **正文**：小字号，轻量，适度留白（建议 18-24pt）
- **原则**：每页文字越少越好，一页一个核心观点

### 配色
- **主色调**：黑 `#1D1D1F` + 白 `#FFFFFF` + 灰 `#86868B`
- **强调色**：仅选择一个，全程保持一致
  - 推荐：苹果蓝 `#0071E3` 或深紫 `#5856D6`
- **禁止**：多种鲜艳颜色混用、渐变文字

### 图形元素
- **图标**：简洁几何线条图标（SF Symbols 风格）
- **容器**：圆角卡片（8-16px 圆角），轻微阴影
- **线条**：细线分隔，颜色为浅灰
- **图表**：扁平化设计，避免 3D 效果

### 布局原则
- **大量留白**：内容不超过页面 60%
- **强对比**：标题与背景形成鲜明对比
- **对齐**：严格的网格对齐，视觉平衡
- **层次**：通过字号和颜色深浅建立信息层级

### 代码块样式
- **背景**：浅灰 `#F5F5F7` 或白色带细边框
- **字体**：SF Mono / JetBrains Mono
- **语法高亮**：低饱和度配色，与整体风格协调

### 交互要求
- 幻灯片之间平滑过渡动画（淡入淡出或滑动）
- 第 8-10 页为交互式 Demo 对比区域（详见下方说明）

---

## 交互式 Demo 对比区域（关键功能）

在实战演示环节（第 8-10 页），我需要在 slides 中**嵌入可交互的前端 Demo 对比**。

### 功能需求
1. **并排展示**：左右或多栏布局，同时展示 2-3 个框架的 UI 效果
2. **模拟交互**：每个框架的 Demo 都应该是可交互的（滑动条可拖动、下拉菜单可选择、按钮可点击）
3. **框架切换**：顶部 Tab 栏可切换不同框架（Gradio / Streamlit / React 等）
4. **代码展示**：每个 Demo 旁边可展开/折叠对应的代码片段

### Demo 任务：AI 算力成本模拟器
所有框架都实现同一个任务，便于对比：
- **输入**：GPU 数量滑动条（1-8）、使用时长滑动条（1-24 小时）、模型选择下拉菜单（GPT-4 / Claude 3.5 / Gemini 1.5）
- **输出**：三个指标卡片（总价、平均单价、风险等级）+ 一个简单的折线图
- **交互**：点击"计算"按钮后显示结果，有 1-2 秒的 loading 动画

### 实现方式
由于无法真正运行 Gradio/Streamlit，请**模拟各框架的 UI 风格**：
- **Gradio 风格**：橙色主题，圆角输入框，标准 Hugging Face 样式
- **Streamlit 风格**：白色背景，红色强调，侧边栏布局
- **React 风格**：暗色主题（可选），精致 Tailwind 风格，磨砂玻璃效果
- **NiceGUI 风格**：Material Design 风格，蓝色主题
- **Django 风格**：传统后台管理界面风格，绿色按钮
